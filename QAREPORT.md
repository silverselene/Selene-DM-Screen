## вњ… Status update вЂ” 2026-06-30 (remaining рџџЎ batch вЂ” findings #21вЂ“#24)

Closed the last four рџџЎ Medium findings вЂ” the open set is now **0 рџ”ґ / 0 рџџ  / 0 рџџЎ / 21 рџџў**:

- **#21 вЂ” Grid resize silently drops widgets.** `handleGridResize` no longer flattens spans and re-pads blindly. It re-packs the real (non-`empty`) widgets into the new grid with a first-fit placer that preserves each widget's `colSpan`/`rowSpan` where it fits (falling back to 1Г—1 before discarding), keeping the `null` span-placeholder bookkeeping consistent. The icon-only Add and Reset buttons gained `title="Add a combatant"` / `title="Reset encounter"`.
- **#23 вЂ” `BASE_PATH` had no compose build-arg.** Dockerfile declares `ARG BASE_PATH=/` + `ENV BASE_PATH=${BASE_PATH}` so the build (which already reads `process.env.BASE_PATH`) honors it; `docker-compose.yml` forwards it via `build.args: BASE_PATH: ${BASE_PATH:-/}`.
